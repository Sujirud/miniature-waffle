การประยุกต์ใช้การคูณเมทริกซ์ของ Strassen สำหรับแก้ปัญหาทางธุรกิจ

ปัญหาทางธุรกิจ

ปัญหาในการสร้างเครื่องคำนวณ
- การจัดการอินพุต : การตรวจสอบให้แน่ใจว่าเครื่องคำนวณประมวลผลอินพุตของผู้ใช้ได้อย่างถูกต้อง โดยเฉพาะอย่างยิ่งสำหรับนิพจน์ที่ซับซ้อน
- การจัดการข้อผิดพลาด : การป้องกันความผิดพลาดหรือผลลัพธ์ที่ไม่ถูกต้องอันเนื่องมาจากการหารด้วยศูนย์ ไวยากรณ์ที่ไม่ถูกต้อง ฯลฯ
- การออกแบบอินเทอร์เฟซผู้ใช้ : การสร้างเค้าโครงที่เรียบง่ายและใช้งานง่าย
- การขยายฟังก์ชัน : การรองรับคุณสมบัติขั้นสูง เช่น ฟังก์ชันทางวิทยาศาสตร์ การติดตามประวัติ หรือการแปลงสกุลเงิน อาจมีความซับซ้อน
- ประสิทธิภาพ : การรับรองการคำนวณที่รวดเร็วและแม่นยำ โดยเฉพาะอย่างยิ่งกับข้อมูลขนาดใหญ่หรือซับซ้อน

Algorithm สำหรับการเปรียบเทียบ - การคูณเมทริกซ์ปกติ

วิธีมาตรฐานสำหรับการคูณเมทริกซ์ขนาด n x n สองเมทริกซ์ จะทำการคำนวณแต่ละองค์ประกอบของเมทริกซ์ผลลัพธ์โดยใช้ผลคูณดอทของแถวจากเมทริกซ์แรกและคอลัมน์จากเมทริกซ์ที่สอง กระบวนการนี้ต้องการการคูณ n3 ครั้ง และการบวก n3 - n2 ครั้ง ซึ่งนำไปสู่ Time complexity ที่ O(n3) สำหรับเมทริกซ์ขนาดเล็ก วิธีนี้ตรงไปตรงมาและมีประสิทธิภาพ อย่างไรก็ตาม เมื่อขนาดเมทริกซ์เพิ่มขึ้น ซึ่งพบได้บ่อยในการประยุกต์ใช้ข้อมูลขนาดใหญ่ เช่น Meachine learning หรือการจำลองทางการเงิน จะส่งผลให้มีค่าใช้จ่ายในการคำนวณเพิ่มขึ้นอย่างมาก

ตัวอย่างเช่น การคูณเมทริกซ์ขนาด 1000 x 1000 สองเมทริกซ์ต้องใช้การคูณประมาณหนึ่งพันล้านครั้ง ซึ่งอาจใช้เวลานานบนฮาร์ดแวร์มาตรฐาน ความไม่มีประสิทธิภาพนี้สามารถทำให้กระบวนการทางธุรกิจที่สำคัญล่าช้า ความต้องการวิธีที่เร็วขึ้นได้ผลักดันให้เกิดการพัฒนาอัลกอริทึม เช่น อัลกอริทึมของ Strassen เพื่อช่วยลดภาระการคำนวณนี้

ขั้นตอนการคูณเมทริกซ์
1. ขนาดของเมทริกซ์ต้องเข้ากันสำหรับการคูณ
ถ้ามีเมทริกซ์ A ขนาด m × n (m แถว และ n คอลัมน์)
และเมทริกซ์ B ขนาด n × p (n แถว และ p คอลัมน์)
จะสามารถคูณเมทริกซ์สองตัวนี้ได้ก็ต่อเมื่อจำนวนคอลัมน์ของ A เท่ากับจำนวนแถวของ B เท่านั้น
ผลลัพธ์ที่ได้จะเป็นเมทริกซ์ C ขนาด m × p

2. วิธีการคูณเมทริกซ์
แต่ละสมาชิกในเมทริกซ์ผลลัพธ์ C คำนวณโดยหาผลคูณจุด :
- ใช้ตัวเลขจากแถวในเมทริกซ์ A และตัวเลขจากคอลัมน์ในเมทริกซ์ B
- คูณสมาชิกจากลำดับที่ตรงกัน
- นำผลคูณทั้งหมดมาบวกกัน
ดังนั้น สมาชิก cij = (แถวที่ i ของ A) • (คอลัมน์ที่ j ของ B)

ตัวอย่างการคูณเมทริกซ์ขนาด 2 * 2 กับเมทริกซ์ขนาด 2 * 2

"ผลคูณจุด" คือการคูณสมาชิกที่ตรงกัน จากนั้นนำไปบวกกัน :
(1 , 2) • (5 , 7) = 1 × 5 + 2 × 7 = 19
จับคู่สมาชิกตัวแรก (1 และ 5) คูณพวกมันเช่นเดียวกันกับสมาชิกตัวที่ 2 (2 และ 7) และสุดท้ายรวมผลคูณเข้าด้วยกัน

สำหรับตำแหน่งอื่นๆ ก็สามารถทำได้ในรูปแบบเดียวกัน :
แถวที่ 1 คอลัมน์ที่ 2 = (1 , 2) • (6 , 8) = 1 × 6 + 2 × 8 = 22
แถวที่ 2 คอลัมน์ที่ 1 = (3 , 4) • (5 , 7) = 3 × 5 + 4 × 7 = 43
แถวที่ 2 คอลัมน์ที่ 2 = (3 , 4) • (6 , 8) = 3 × 6 + 4 × 8 = 50

import time
import numpy as np

A = np.loadtxt("matrix_A.txt", dtype=int)
B = np.loadtxt("matrix_B.txt", dtype=int)
n = len(A)
result = [[0 for _ in range(n)] for _ in range(n)]  
            
start = time.time()
            
for i in range(n):
    for j in range(n):
        for k in range(n):
            result[i][j] += A[i][k] * B[k][j]
            
time_spent = time.time() - start
            
print(f"{time_spent:.4f} Second")

Algorithm ของ Strassen

อัลกอริทึมของ Strassen ซึ่งนำเสนอโดย Volker Strassen ในปี 1969 ปฏิวัติการคูณเมทริกซ์ด้วยการลดจำนวนการดำเนินการที่จำเป็น ซึ่งแตกต่างจากวิธีมาตรฐานซึ่งใช้การคูณแปดครั้งในการคำนวณผลคูณเมทริกซ์ 2x2 อัลกอริทึมของ Strassen ให้ผลลัพธ์เดียวกันด้วยการคูณเจ็ดครั้งและการบวก/ลบเพิ่มเติม การลดลงนี้วนซ้ำโดยการแบ่งเมทริกซ์ขนาดใหญ่เป็นเมทริกซ์ย่อยที่เล็กกว่า ส่งผลให้ Time complexity อยู่ที่ประมาณ O(n2.81) ซึ่งดีขึ้นอย่างมีนัยสำคัญเมื่อเทียบกับ O(n3)

กลยุทธ์แบ่งแยกและเอาชนะของอัลกอริทึมทำงานดังต่อไปนี้ :
1. แบ่งเมทริกซ์ n x n แต่ละเมทริกซ์เป็นเมทริกซ์ย่อย n/2 x n/2 จำนวนสี่เมทริกซ์
2. สร้างเมทริกซ์ใหม่ 7 รายการจากเมทริกซ์ย่อย :

p1 = a(f - h)
p2 = (a + b)h
p3 = (c + d)e
p4 = d(g - e)
p5 = (a + d)(e + h)
p6 = (b - d)(g + h)
p7 = (a - c)(e + f)

3. รวมผลลัพธ์เพื่อสร้างเมทริกซ์สุดท้ายโดยใช้การบวกและการลบ :

C11 = p5 + p4 - p2 + p6
C12 = p1 + p2
C21 = p3 + p4
C22 = p1 + p5 - p3 - p7

ตัวอย่างการคูณเมทริกซ์ขนาด 2 * 2 กับเมทริกซ์ขนาด 2 * 2

1. แบ่งเมทริกซ์ :

ได้เมทริกซ์ย่อยจากเมทริกซ์ A คือ 1 , 2 , 3 , 4
และเมทริกซ์ย่อยจากเมทริกซ์ B คือ 5 , 6 , 7 , 8

2. สร้างเมทริกซ์ใหม่ 7 รายการจากเมทริกซ์ย่อย :

p1 = a(f - h) = 1(6 - 8) = -2
p2 = (a + b)h = (1 + 2)8 = 24
p3 = (c + d)e = (3 + 4)5 = 35
p4 = d(g - e) = 4(7 - 5) = 8
p5 = (a + d)(e + h) = (1 + 4)(5 + 8) = 65
p6 = (b - d)(g + h) = (2 - 4)(7 + 8) = -30
p7 = (a - c)(e + f) = (1 - 3)(5 + 6) = -22

3. รวมผลลัพธ์เพื่อสร้างเมทริกซ์สุดท้าย :

C11 = p5 + p4 - p2 + p6 = 65 + 8 -24 +(-30) = 19
C12 = p1 + p2 = -2 + 24 = 22
C21 = p3 + p4 = 35 + 8 = 43
C22 = p1 + p5 - p3 - p7 = -2 + 65 -35 -(-22) = 50

import time
import numpy as np
            
def strassen_multiply(A, B):
    n = A.shape[0]

    if n <= 2:
        return A @ B

    mid = n // 2
    A11 = A[:mid, :mid]
    A12 = A[:mid, mid:]
    A21 = A[mid:, :mid]
    A22 = A[mid:, mid:]
    
    B11 = B[:mid, :mid]
    B12 = B[:mid, mid:]
    B21 = B[mid:, :mid]
    B22 = B[mid:, mid:]

    P1 = strassen_multiply(A11, B12 - B22)
    P2 = strassen_multiply(A11 + A12, B22)
    P3 = strassen_multiply(A21 + A22, B11)
    P4 = strassen_multiply(A22, B21 - B11)
    P5 = strassen_multiply(A11 + A22, B11 + B22)
    P6 = strassen_multiply(A12 - A22, B21 + B22)
    P7 = strassen_multiply(A11 - A21, B11 + B12)
    
    C11 = P5 + P4 - P2 + P6
    C12 = P1 + P2
    C21 = P3 + P4
    C22 = P1 + P5 - P3 - P7

    return np.vstack((np.hstack((C11, C12)), np.hstack((C21, C22))))  

A = np.loadtxt("matrix_A.txt", dtype=int)
B = np.loadtxt("matrix_B.txt", dtype=int)

start = time.time()
            
result = strassen_multiply(A, B)
            
time_spent = time.time() - start
            
print(f"{time_spent:.4f} Second")

การประยุกต์

ผลการทดลอง
จากการทดลองนี้ได้เปรียบเทียบประสิทธิภาพของการคูณเมทริกซ์ปกติกับการใช้อัลกอริทึมของ Strassen โดยบันทึกเวลาที่ใช้สำหรับแต่ละวิธี และได้ผลลัพธ์ดังนี้ :
การคูณเมทริกซ์ปกติ : 4.7429 วินาที
การคูณเมทริกซ์โดยใช้อัลกอริทึมของ Strassen : 2.7207 วินาที

พบว่าอัลกอริทึมของ Strassen มีประสิทธิภาพเหนือกว่าวิธีดั้งเดิมอย่างเห็นได้ชัด โดยลดเวลาในการคำนวณลงประมาณ 42.67% ซึ่งแสดงให้เห็นถึงประสิทธิภาพของอัลกอริทึมของ Strassen สำหรับการคูณเมทริกซ์

ประโยชน์ทางธุรกิจ
ข้อได้เปรียบหลักของอัลกอริทึมของ Strassen คือความสามารถในการลดเวลาการคำนวณสำหรับการคูณเมทริกซ์ขนาดใหญ่ ซึ่งจะทำให้สามารถประมวลผลงานที่สำคัญต่อธุรกิจได้เร็วขึ้น ประสิทธิภาพดังกล่าวช่วยให้ธุรกิจสามารถจัดการชุดข้อมูลขนาดใหญ่ เรียกใช้แบบจำลองที่ซับซ้อนยิ่งขึ้น และตัดสินใจตามข้อมูลได้เร็วขึ้น ตัวอย่างเช่น ใน machine learning เวลาในการฝึกที่สั้นลงสามารถเร่งการใช้งาน predictive models ได้ ทำให้ธุรกิจมีความได้เปรียบในการแข่งขัน นอกจากนี้ อัลกอริทึมของ Strassen ยังถูกผสานเข้ากับไลบรารีตัวเลขที่ใช้กันอย่างแพร่หลาย เช่น BLAS ซึ่งหมายความว่าธุรกิจจะได้รับประโยชน์โดยไม่จำเป็นต้องนำไปใช้โดยตรง ความสามารถในการเข้าถึงนี้ทำให้ใช้งานได้จริงในการประยุกต์ใช้ต่างๆ ตั้งแต่ซอฟต์แวร์ทางการเงินไปจนถึงแพลตฟอร์มการวิเคราะห์ข้อมูล

อย่างไรก็ตาม มีข้อแลกเปลี่ยนที่ต้องพิจารณา :
• การใช้หน่วยความจำ : ลักษณะการวนซ้ำของอัลกอริทึมของ Strassen ต้องใช้หน่วยความจำเพิ่มเติมเพื่อจัดเก็บเมทริกซ์ย่อยระดับกลาง ซึ่งอาจเป็นข้อจำกัดในระบบที่มีทรัพยากรจำกัด
• จุดได้เปรียบ : ข้อดีของอัลกอริทึมนั้นเด่นชัดที่สุดสำหรับเมทริกซ์ที่มีขนาดใหญ่กว่า 100x100 โดยการศึกษาวิจัยบางกรณีแนะนำให้ใช้เกณฑ์ 1000x1000 สำหรับฮาร์ดแวร์สมัยใหม่ สำหรับเมทริกซ์ขนาดเล็กวิธีมาตรฐานอาจเร็วกว่าเนื่องจากมีค่าใช้จ่ายทางอ้อมน้อยกว่า
